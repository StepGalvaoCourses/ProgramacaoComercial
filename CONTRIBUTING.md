# Guia de Contribuição

Obrigado por considerar contribuir para este repositório! Este guia ajudará você a fazer contribuições de forma efetiva.

## Como Contribuir

### Para Estudantes

1. **Reportar Erros**
   - Use as Issues do GitHub para reportar erros no material
   - Descreva claramente o problema encontrado
   - Inclua exemplos quando possível

2. **Sugerir Melhorias**
   - Abra uma Issue descrevendo sua sugestão
   - Explique como a melhoria beneficiaria o curso

3. **Compartilhar Soluções Alternativas**
   - Você pode compartilhar diferentes abordagens para resolver exercícios
   - Use Pull Requests para propor suas soluções
   - Inclua comentários explicando sua lógica

## Boas Práticas

### Código

- Use nomes de variáveis descritivos
- Adicione comentários para explicar lógica complexa
- Siga as convenções de estilo da linguagem
- Mantenha o código limpo e legível

### Commits

- Escreva mensagens de commit claras e descritivas
- Use verbos no imperativo (ex: "Adiciona", "Corrige", "Atualiza")
- Faça commits pequenos e focados

### Pull Requests

1. Faça um fork do repositório
2. Crie uma branch para sua contribuição
   ```bash
   git checkout -b minha-contribuicao
   ```
3. Faça suas alterações
4. Commit suas mudanças
5. Push para sua branch
6. Abra um Pull Request

## Código de Conduta

- Seja respeitoso com todos os participantes
- Aceite feedback construtivo
- Foque em aprender e ajudar outros a aprender
- Mantenha um ambiente colaborativo e inclusivo

## Dúvidas?

Se tiver dúvidas sobre como contribuir, abra uma Issue ou entre em contato através dos canais do curso.

## Licença

Ao contribuir, você concorda que suas contribuições serão licenciadas sob a mesma licença do projeto.
